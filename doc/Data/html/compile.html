CHAPTER 4 Compiling Ingestors

The ingestors included in the Zebra distribution are normally not compiled by the default configuration. This chapter explains how to change the configuration to select the compilers you need to build and install. This section does not cover the procedures necessary for writing a new ingestor; see instead the Zebra Developer's Guide.

The next section of config.h specifies the ingest programs to build during compilation. These programs are not likely to ever be needed except for sites which are running Zebra in real-time. If down the road you discover a need for a particular ingest module, it is easier to compile that module individually at that time. To avoid compiling all of the ingest modules for now, leave this line unchanged. 

#define BuildIngest NO

This will disable compilation of all of the ingest modules under the directory /zeb/src/ingest. To enable the compilation of an ingest module, the BuildIngest symbol must first be defined as YES. Then the determination of whether an ingest program is compiled rests with the setting of that ingest module's specific build symbol in the following lines of config.h. For example, the satellite ingest programs will be compiled if BuildIngest is YES and BuildSatelliteIngest is YES. If BuildIngest is NO, then the definition of BuildSatelliteIngest is ignored and the satellite ingest programs will not be compiled.

Caution: Some of the ingest programs require specialized libraries or presently only compile on certain machines. For HP-UX, if BuildIngest has been defined as YES, then the following symbols should be defined to NO. 

# define BuildIngestScheduler NO /* Sun only */
# define BuildRadarIngest NO /* Sun only */
# define BuildSLGrabber NO /* Sun only */
# define Build_TRMM_Rain NO /* Sun only */

On HP-UX, all of other ingest modules will compile. Under SunOS and Solaris, all of the ingest modules---including the four above---can be compiled and installed given that any necessary libraries exist on the system.

In most cases the only documentation for an ingest program is a usage message and the code itself. If you think you need a particular ingest program, be prepared to wrestle with it a little bit to get it to meet your application. Or, feel free to request assistance from one of the persons listed in the "Whom to Contact" section of the Installation Manual. For more detailed information about the ingest modules, such as descriptions of each, see Section 3.2, "Ingest Modules in Release 4.2".
