APPENDIX B Summary of the Installation Steps

This appendix briefly outlines the steps in compiling the source distribution of Zebra. It is intended as a general guideline for those who are familiar with the installation of Zebra from previous attempts, or for those who are familiar with UNIX software installations in general. If you would like to start with the complete and detailed instructions, see the beginning of this manual.

Compiling and installing Zebra on a supported system is generally very straightforward. You need to know the location of the third-party libraries and include files---described in Section 7, "Acquiring Third-Party Software", the correct C and FORTRAN compiler commands, and the FORTRAN libraries required by the FORTRAN compiler. Then follow the instructions below. Note that most make programs will work. In particular, the standard system make will work as well as GNU make. The make process will use whichever make program appears first in your path.

  1. Untar the source distribution into a directory with enough space for the source code, object files, and installed files---about 60 megabytes, or as much as 100 megabytes if compiling with OpenWindows libraries or linking statically.

  2. From the top of the Zebra source tree, change to the directory config/project.

  3. Edit config.h according to the instructions within the file.

  4. If you do not have the RDSS libraries or wish to install them from a combined Zebra/RDSS distribution, define BuildRDSS to YES in config.h.

  5. Make sure the environment variable LD_LIBRARY_PATH is not set. 



unsetenv LD_LIBRARY_PATH



    

  1. Return to the top directory of the source tree. Then, on all systems except Solaris, run




make World >& Make.World &



    

  1. On Solaris systems, run




make BOOTSTRAPCFLAGS=-Dsolaris World >& \
  Make.World &



    

  1. Check the log file for errors. Errors in the log file likely indicate a need to adjust config.h. See Section 10, "In Case of Errors" for troubleshooting suggestions. If necessary, re-compile from the top-level directory with the same command used for the initial build, given in steps (7) and (8).

    

  2. After a successful build, check the operating system issues mentioned in Section 12, "Tuning the Operating System".

    

  3. Introduce yourself to Zebra with the Introducing Zebra manual.





If these instructions are not clear, or problems occur, follow the more detailed instructions given in the full Installation manual.
